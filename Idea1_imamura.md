# 概要
- 手書きの文字をアップロードすると、それを学習して他の文字についてもその人の手書き文字で表示できるようにする
- 生成した手書き文字の出力方式は複数選べるようにしたい


# ユーザー利用の流れイメージ
## 手書き文字アップロード
- スマホからのアップロードを想定（PCでスキャンするものできるようにはしたい)
- アップロードの方法
	- 専用の用紙を印刷してそれに文字を記入 -> 撮影
		- せっかくスマホを使っているのにここだけ紙なのがネック
		- 撮影のしたデータを数値に置き換える処理が発生
		- 一方、手書き感は出る
	- スマホアプリ上で記入（タッチペンなど？）
		- そもそもスマホは指で操作するから...
		- データ化は簡単
- アップロードする文字の案
	- ユーザーの自由にする
	- 指定したものを書かせる（事前のテストで一番特徴をつかみやすいもの）

### 入力言語について
- ゆくゆくはどの言語でも対応できるようにしたい
- とりあえずは、言語を選ばせる（国連で使われている言語くらいはサポートしたい...？）

## ユーザーの文字を学習
- どのような特徴量を使うと良いかはメタデータの学習を通じて強化していくスタイルにしたい -> トライアルサイトをつくるなり、毎回フィードバックをもらうなり

## 生成文字列を出力
- ユーザーが入力していない文字で文章を構成して、文字間隔などがうまくいっているかフィードバックをもらう
- 違和感のあるところがあれば指摘してもらって、即座に修正する
	- これを通じてメタデータを収集、学習する

### 出力形式
- 各OSで使える文字ファイル形式
	- mac
	- windows
- ユーザーの指定した文字列を単に出力する
	- 画面上
	- pdf

### 出力文字の定義
- ユーザーが指定した文字列を出力するだけなら問題なし
- 全て出力する必要があるとき...？
	- DBに一覧を持っておく？
	- 一般的に使われる文字がどこかに定義されていたりする...？

### 出力言語
- まずは入力言語と同じもの
- ゆくゆくはどの言語にも変換できるようにしたい
